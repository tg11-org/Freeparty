Changelog

A running log of major updates and improvements to Freeparty.

2026-04-23 — Security Hardening: oEmbed XSS, SSRF Guards, DM Dedup, Safer Defaults

  • Hardened link preview embeds: provider oEmbed HTML is now sanitized with a strict iframe allowlist before it is stored/rendered.
  • Improved SSRF protections for remote fetches: URLs are validated for allowed schemes, DNS A/AAAA resolution, and private/reserved IP ranges.
  • Federation inbound/outbound fetch now validates the final URL after redirects and enforces the expected instance domain.
  • Direct message threads are now de-duplicated at the database layer using a canonical participant key to prevent race-created duplicate conversations.
  • Encrypted DM send rate-limit check is now enforced inside a transaction lock to prevent concurrent bypass.
  • Production bootstrap defaults tightened: setup script now writes HTTPS-only trusted origins and preserves existing DB credentials; local compose uses env-driven Postgres credentials.
  • Removed deprecated Django setting SECURE_BROWSER_XSS_FILTER in production configuration.
  • Performance improvements: bookmarks visibility avoids per-item permission queries, and unread DM conversation count is computed via DB-side EXISTS checks.
  • Email reliability update: permanent SMTP recipient rejects (for example unknown mailbox) are now treated as terminal non-retriable task failures, reducing noisy retries during signup/password email flows.
  • Account UX fix: /accounts/verify/resend/ now handles GET requests gracefully with a redirect and helpful message instead of showing a method error/blank page.
  • Private profile UX update: actor detail now renders the same restricted profile page used for block scenarios instead of returning a hard 404 for private-account access denial.
  • Private profile restricted page now includes a Request to follow action (plus pending/cancel states) so follow-request flows can be initiated directly when profile content is hidden.
  • Notifications now show better post context when a post body is empty: they fall back to attachment caption first, then to the post ID label.
  • Fixed reopening legacy direct-message threads on PostgreSQL by removing an invalid SELECT FOR UPDATE + DISTINCT query combination from the blank-key compatibility lookup.

2026-04-22 — Security Hardening, Icon Controls & Further UI Polish

  • Added server-side CSS injection sanitization for custom theme color inputs — hex fields are validated against a strict pattern and the gradient field now uses a character allowlist, blocking any attempt to inject </style>, <script>, or other hostile markup.
  • Added client-side sanitization pass on custom theme tokens before they are written to CSS custom properties, as an additional defense-in-depth layer.
  • Added Icons toggle in the accessibility bar to cycle between Auto (follows theme), Light (always inverted, ideal for AMOLED/dark themes), and Dark (always original) SVG icon styles.
  • Added JetBrains Mono as a fourth font option in the font cycle, joining Dyslexic, Atkinson, and IBM Plex.
  • Fixed Show follower list and Show following list privacy toggles not rendering as styled switch controls on the profile edit page — they now match all other toggle inputs.
  • Fixed custom theme colors entered without a leading # or with a trailing ; not applying correctly — both are now auto-corrected on save and at page paint time.
  • Improved post action button row: all buttons now share a consistent height regardless of whether they contain an icon, a count, or both, and clicking no longer causes a layout shift.
  • Added inline tip on the profile edit page reminding users to omit the trailing semicolon from hex color values.
  • Styled file picker buttons, unfollow button on the Following page, and account management buttons (Regenerate recovery codes, Disable 2FA) to match the rest of the UI.
  • Added compact linked color pickers next to custom theme hex inputs on profile edit, keeping the control visually small while syncing with text values for copy/paste workflows.
  • Added custom theme import/export tools on profile edit with YAML-style sharing text, including copy/download export and tolerant import parsing for keys like accent-alt and bare gradient lines.
  • Promoted the compact floating Top button into a global control in the base layout, shown on long/scrollable pages after scrolling down.
  • Made post reactions mutually exclusive: setting Like now clears Dislike and vice versa, including immediate async counter/state updates.
  • Fixed toggle-switch rendering regressions by unifying post creation toggles with the shared global toggle markup/CSS and hardening selector logic in the base stylesheet.
